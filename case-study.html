Case Study

Real-Time Crowd Intelligence

Designing a real-time crowd intelligence system for smarter commuting decisions

UI/UX Research Dashboard Data Visualization

Overview

RTPTCIS is a real-time system built to improve public transport experiences by introducing real-time crowd visibility. The system helps commuters choose better travel options while giving administrators tools to monitor passenger flow and system performance.

My Role

I led the end-to-end UX design process, including research synthesis, user journey mapping, wireframing, and high-fidelity UI design. I focused on translating complex real-time data into clear, actionable visual insights for faster decision-making.

Problem

Most commuters rely only on train schedules without knowing actual crowd conditions. This often results in overcrowded journeys, wasted time, and stressful decision-making. There is a clear gap between available transport data and how easily users can use it.

Research & Insights

  • Commuters rely heavily on schedules but lack visibility into crowd density
  • Peak-hour uncertainty leads to stress and poor travel decisions
  • Users prefer quick visual cues over reading detailed information
  • Comparing multiple train options is a common behavior

Pain Points

No real-time crowd info
Hard to compare trains
Time pressure
Confusing apps

User Persona

Rahul, 24 — Office Commuter

Behavior: Travels daily during peak hours using local trains

Goal: Find less crowded trains quickly

Pain: No real-time visibility of crowd conditions

Frustration: Ends up in overcrowded trains due to lack of data

Goal

  • Enable users to identify less crowded trains instantly
  • Reduce effort required to make travel decisions
  • Present real-time data in a simple and intuitive format

UX Strategy

  • Speed-first design for quick decisions
  • Use visuals instead of heavy text
  • Enable easy comparison of options
  • Maintain consistency across screens

Designed a clean, data-driven dashboard that translates complex real-time crowd information into intuitive visual indicators, enabling users to compare options quickly and make confident travel decisions.

Key Learnings

Simplicity Wins

Users prefer fast and clear decisions over complex features.

Visual > Text

Color indicators and charts improve understanding instantly.

User Stress Matters

Design should reduce pressure during peak-time decisions.

Iteration is Key

Testing and refining improves usability significantly.
